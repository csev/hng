Setting up a Network with Two Windows-98 Computers

This case study assumes that you have two computers and would like to connect them together to share files and printers. This is a very simple operation, and while this tutorial has a number of steps it can be done in less than an hour.

This tutorial does not include the physical installation of the hardware. It assumes that you have installed network cards in both computers, connected them using two cables and a hub (make sure the hub is plugged in).

This tutorial begins when one of the computers is booted up and detects the newly inserted network card.

Windows-98 has detected the card and will search for drivers. At this point, you can press Cancel to skip this process and continue booting the system. You can add the drivers later using the control panel or at a later re-boot.

In this example, we will press Next.

Generally you will allow Windows to search for the best drivers for the hardware. You will later have a chance to install drivers from the floppy disk.

If you choose the second option, you will select the hardware from a list. You will also have a chance to install your drivers from a disk using the second option.

We have our driver on a floppy disk, so we check the floppy-disk option.

Sometimes a driver is located in a subdirectory on the floppy (or CD-ROM) so you must use the Browse option to select the proper subdirectory. Often directories are of the form A:\WIN98\ or A:\DRIVERS\WIN98\. Most vendors provide floppies or CD-ROM directory structures which can automatically be read by Windows.

Sometimes it is helpful to look through the media on another system to get a sense of the directory structure. You are looking for files that look like LNE100TX.INF. These INF files contain the information indicating which files must be installed for which pieces of hardware.

This floppy has the INF file in the root directory so it is automatically found.

If no driver is found, press the Back button and try again. Ultimately you may have to press Cancel if you simply cannot find the driver.

In some cases, Windows may find a built-in driver for a card. Some older cards have drivers included in the Windows-98 distribution media. Also some newer cards are designed to emulate older cards such as the NE2000 card. If the card did not come with a driver disk at all, it is likely that it emulates some card that Windows-98 already supports.

In addition to the drivers for the card, Windows-98 notices that this is the first network card installed on the system and must install certain networking components to support the card. This includes the TCP/IP software.

So you must insert the Windows-98 CD and press OK.

Copying the files.

During the copy process, the installer may get confused and raise dialogs. In this case, after copying some files from the Windows-98 CD-ROM, it is looking for the original driver from the floppy and cannot find it. Simply re-select the A:\ path from the drop down box.

This type of dialog may come up several times as the installer must switch media. Simply read the directions and help the installer find the media. If you make a mistake and select the wrong media, it will tell you and give you another chance. If it is looking for something and you cannot help it find it, you may have to press Cancel.

Sometimes you can find some components in C:\WINDOWS\SYSTEM\ or C:\WINDOWS\OPTIONS\CABS\ - this is especially true if you are reinstalling something and the files have already been installed once.

This is a good indication. With your help, Windows has found all of the necessary drivers.

Press Finish.

The first of several necessary restarts.

Once the system is back up and we have logged in properly, we go to Start | Settings | Control Panel | Network and see this screen.

A number of components have been installed. We see our new adapter and the TCP/IP protocol. The TCP/IP protocol has been "bound" to both the dial-up adapter and our new network card.

This configuration would make the most sense in a business environment when this computer is connected to a LAN which is already connected to the Internet.

You can explore these components by highlighting them and clicking Properties for each.

Because we are going to be operating in a home network environment and only do file and print sharing, we are going to remove the TCP/IP protocol bound to the Network adapter.

In a more complex setup, we would leave this binding. If we want it back, we can always add it later.

So we select it and press Remove.

Next we must add a protocol for file and print sharing. On the Network control panel we press Add and this screen comes up.

We select Protocol and press Add.

Navigate until the vendor is Microsoft and the protocol is NetBEUI.

NetBEUI is a file and print sharing protocol developed by Microsoft which requires no configuration. It is a very fast and efficient protocol which is very easy to use for small workgroups.

In business environments, either TCP/IP or Novell protocols may be used for file sharing, but they require more configuration.

Once you press OK be patient, after a few seconds, you will be brought back to the control panel screen.

It has automatically added NetBEUI to our dialup adapter. There is no reason to have this, so we remove it.

This is our final configuration of network protocols.

Make sure the login is set to Client for Microsoft Networks - unless this is set, the system will not be able to participate on the network.

When this is all set, press OK.

Again, our new configuration will need some more Windows-98 files installed.

The files will be installed and the system will reboot.

When the computer reboots, you will be presented with a login screen.

There are two different screens used for log-in. A "Windows" login (smaller screen with a windows logo) and a "Network" login. You need to see the network login. If you don't, go to the Network Control Panel and make Client for Microsoft Networks the default login.

Also make sure to press OK or Enter at this screen - not Escape. If you press Escape, you will not be logged in and file/print sharing will not work.

If you feel brave and want to alter the log-in process, see the Frequently Asked Questions for this tutorial.

A quick way to get the Network Properties is to right-click on Network Neighborhood and select Properties.

Back in the network properties, we select Identification. We must select a network name for the computer, select the workgroup shared by all of the computers, and put in an optional description.

Computer names must be unique within the workgroup.

Under access control, we leave the default set to Share-level control.

this allows you to set passwords based on what is being shared (disk or printer) rather than on some user or group structure. The second option is typically used in a business environment running an NT Domain.

Now we must turn on file and printer sharing so we press the button File and Print Sharing. Until we indicate that we have something to share, our computer will not appear when we browse the network.

We indicate that we will be sharing both files and printers.

This does not actually permit others to use our resources, it just installs the necessary software so we can configure what and how we will share resources later.

Once these are checked press OK and press OK again in the network control panel. You will be prompted for the CD-ROM and the system will be rebooted again.

Now you will go through the exact same setup on the second system you have connected to the network.

Everything is identical to the first system except the computer name in the Identification section.

Once everything is set, you might want to reboot the both computers one more time.

Now is the moment of truth. On either computer (in this example I will use the downstairs computer). Double click on Network Neighborhood. If all goes well, you will see both computers.

If you cannot browse the network, re-check all the settings on both computers, re-check the hardware (do you have link-lights on the hub), and reboot the computers.

Other than hardware or connection problems, the most common problem is not logging in properly. Having the wrong type of login configured or pressing Escape at the login screen is a very common problem.

If you cannot browse the network, sometimes you can find a computer directly. To do this, right-click on Network Neighborhood, select Find Computer, enter the computer name (upstairs) in the field, and press Find. Sometimes you can communicate with a computer but cannot browse the network.

There is further information in the Frequently Asked Questions supplement for this tutorial.



Before we can access data, we must first give others permission to access the data. So on the downstairs computer, we navigate to our hard drive, right click, and select Sharing.

We indicate that the drive is to be shared, and leave the share name as default. It is possible to create a share name which is not the drive letter, but most people keep the simple default.

You can indicate the read/write passwords. In this example, we are giving full access with a single password.

The little hand icon indicates that the C drive is now shared.

We indicate that the downstairs printer is shared using the same right-click mechanism.

Under the Sharing tab, we accept the default share name and do not put a password on the printer.

The "Sharing Hand" indicates that the printer is now shared.

Moving to the upstairs computer, we log-in and go to network neighborhood. We double click on the downstairs computer to find out what items are available to us on that computer.

We see two shares, a printer and a disk drive. First we will explore the disk drive so we double-click it.

We assigned a password to this share, so we must enter it here. The upstairs system can store this password for later use.

Since we have the right password, we are now browsing the downstairs computer's C: drive over the network. You can interact with this drive using drag and drop, or any way that you would use a local folder.

Sometimes it is more convenienent to make the remote drive into a "virtual drive" so we don't have to navigate each time through the Network Neighborhood each time. To do this, we right click on the share name and select Map Network Drive.

We pick the drive to letter to map, and indicate that we want this drive mapped at every log-in and press OK.

You may have to enter a password, but since we already have entered the password for the drive once, it is mounted without a password.

Looking at My Computer we see the new "virtual" E: drive.

To access the shared printer, while browsing the shares, we right-click on the printer and select Install. This will walk through the process of installing the printer. One nice feature is that the necessary drivers to support the printer are installed across the network.
In this case tutorial, we have walked through the configuration of two computers on a network and setup sharing for files and a printer.

For a typical PC user, this is not much more complicated than installing a modem. There are a few settings which may not be immediately obvious but this tutorial should show those settings.
